Name Web Attacks

SQL Injection

SQL Injection refers to when an attacker is able to inject arbitrary SQL queries by overloading unsanitised input. If a malicious user identifies unsanitied inputs, they can input at semicolon and insert a new query to extract or modify date.
Source:https://blogs.zeiss.com/digital-innovation/en/lets-talk-about-security-sql-injection/

Cross-Site Scripting (XSS)

Custom Javascript is maliciously injected into a users browser session without the knowledge of the user. An instruction to download a script (sometimes in for example the comments section of website). The script can then be used to steal cookies, session tokens, or other sensitive information.
Source: https://www.imperva.com/learn/application-security/cross-site-scripting-xss-attacks/

Cross-Site Request Forgery (CSRF)

CSRF is an attack that is an extension on XSS. When a person tries to do a priveleged action, the attacker is able to intercepts the attack and perform that action on behalf of the user, but modify the paramters in such a way that isn't noticable to the user.
Source: https://www.wallarm.com/what/what-is-cross-site-request-forgery

Denial of Service (DoS)

A DoS attack is an attack that limits the availability of a service to users by overloading the server with requests. This can be done by spamming requests to the server and causing it to crash, or by sending a large amount of data to the server to fill up the servers memory. A particula type of DoS attack is a DDoS attack, which is when multiple devices are used to attack a server.
Source: https://www.cloudflare.com/en-au/learning/ddos/what-is-a-ddos-attack/

Man-in-the-Middle (MitM)

A man in the middle attack is an attack where the victim is tricked into communicating through the attacker, allowing the attacker to manipulate or read the data being sent to or from the victim.
Source: https://www.imperva.com/learn/application-security/man-in-the-middle-attack-mitm/

Phishing

A phishing attack is where a victim is convinced into clicking a link sent to them by a malicious actor, using fear or greed to trick them into thinking the message they're being sent is legitimiate. Once the link is clicked, user may be tricked into downloading malware, or entering credentials into a fake website.
Source: https://www.cloudflare.com/en-au/learning/access-management/phishing-attack/

Brute Force Attack

A brute force attack is where an attack attempts to guess the password for a user by systematically trying username and password guesses. Once the attacker has the correct credentials, they are able to access the account, steal information, manipulate date or change the password to deny the original use access to their account.
Source: https://www.spiceworks.com/it-security/cyber-risk-management/articles/what-is-brute-force-attack/

Remote Code Execution

Remote code execution is where an attacker is able to run code on a victims machine without their knowledge. By getting this arbitrary code to run, Attackers can take over machines, steal data or disrupt service.
Source: https://patchmypc.com/remote-code-execution-vulnerabilty

Directory Traversal

A directory traversal attack is where an attacker is able to access files and directories that are outside of where the web administrator intended, by manipulating the address input whether directly by the path or by some sort of paramter. Attackers can use this access to extract secret or sensitive data.
Source: https://www.fastly.com/learning/what-is-directory-traversal

Cryptojacking

Cryptojacking is an attack where a victim is tricked into running a cryptomining script on their machine when they visit a website. The script runs in the background without their knowledge (a particular type of remote code execution), while cryptocurrency is mined for the attacker.
Source: https://securityintelligence.com/posts/cryptojacking-attacks-whos-mining-on-your-coin/
